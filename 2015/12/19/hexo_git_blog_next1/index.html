hexo搭建github博客(NexT主题)(一)

发表于 2015-12-19   |   分类于 软件及网络应用

〇 前言

这几天在网上搜索了下,但没有找到一篇教程能让新手从零开始搭建一个完整的NexT主题博客.
同时觉得自己也应该尝试记录点东西,无论是加深印象或是日后查找或是学习语言组织,都或多或少是有点好处的.
因此花了一个晚上的时间写了这篇教程.

简单地说下原理:利用hexo将本地的生成的博客内容部署至github代码仓库,再由github自动布局远程页面.

至于为什么要使用github,除了一丁点比格外,最主要的是免费(几年前是每个仓库有300M的免费空间,现在貌似没有明确限制).

所以开始吧!

一. 注册gitHub账号，进入gitHub官网,完成图示流程

  1. 点击
  2. 填写账号(我的是yangzongjie),邮箱,密码
  3. Chosen Free,Fish sigh up

二 登陆邮箱,验证账号

三 gitHub创建仓库（creat repository）

  1. 点击页面右上角“+”，选择“creat repository”
  2. 填写Repository name，格式为“yangzongjie.github.io”；点击Creat repository.
  3. 进入 https://github.com/yangzongjie/yangzongjie.github.io
    点击Settings
  4. 将页面拉到底部，点击”Launch automatic page generator”
  5. 点击Continue to layouts
  6. 点击Publish page

四 安装Homebrew ,Node.js，Git

  1. 切换为RubyGems 镜像源为淘宝。 
$ gem sources --add https://ruby.taobao.org/ --remove https://rubygems.org/
$ gem sources -l
*** CURRENT SOURCES ***

https://ruby.taobao.org

$ gem install rails

  1. 安装Homebrew 

    $ ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

  2. homebrew安装git(若装有xcode，则跳过该步骤) 

    $ brew install git

  3. homebrew安装Node.js 

    $ brew install node

若出现编译问题，建议安装Xcode，启动并进入 Preferences -> Download -> Command Line Tools -> Install 安装命令行工具

五 生成git ssh key

  1. 打开终端，配置用户名，邮箱 

    $ git config --global user.name "yangzongjie"  
$ git config --global user.email "340244120@qq.com"

  2. 删除旧ssh key （第一次配置ssh则跳过该步骤) 

    $ cd ~
$ sudo rm .ssh -r -f

  3. 生成key 

    $ ssh-keygen -t rsa -C “340244120@qq.com”

    连续按3个回车

  4. 将生成的key写入本地文件 

    $ ssh-add id_rsa.pub

  5. 拷贝key，并贴入gitHub
    打开finder，按下command+shift+G,输入~/.ssh 回车，生成key即保存在id_rsa.pub。
    右键用“文本编辑”打开 id_rsa.pub。将里面文字（类似ssh-rsa AAAAB3NzaC1yc…）全部复制。
    打开https://github.com/settings/ssh 设置gitSSHKey
    点击add SSH key;在Key里粘贴之前复制的key。（Title任填）

  6. 验证添加key是否成功 

    $ ssh git@github.com

    若显示 

    $ ssh git@github.com  
    PTY allocation request failed on channel 0`   
    Hi yangzongjie! You've successfully authenticated, but GitHub does not provide shell access.`    
    Connection to github.com closed.

则OK.

六 安装Hexo

1 进入安装文件夹 (如/Users/hijack/Develop/yangzongjie.github.io) 

$ cd /Users/hijack/Develop/yangzongjie.github.io

2 安装Hexo到当前目录 

$ npm install -g hexo-cli   
$ hexo init         
$ npm install

此时文件夹内目录日下
.
├── _config.yml
├── package.json
├── scaffolds
├── source （存放本地资源；子文件件_post用于存放文章.）
└── themes (存放主题)

七 安装NexT主题

  1. 下载主题(位于NexTG gitHub的pisces分支)

    $ cd /Users/hijack/Develop/yangzongjie.github.io
    $ git clone -b pisces https://github.com/iissnan/hexo-theme-next themes/next

  2. 站点配置文件&主题配置文件 （==冒号后面接空格==）

    ==站点配置文件==位于/Users/hijack/Develop/yangzongjie.github.io/_config.yml。
    ==主题配置文件==位于/Users/hijack/Develop/yangzongjie.github.io/themes/next/_config.yml。

  3. 打开 站点配置文件（修改语言、主题等）
    3.1 找到 theme 字段，并将其值更改为 next。（theme: next）
    3.2 简体中文 language: zh-hans
    3.3 设置头像： 在空白行增加avatar= https://头像地址
    3.4 设置github连接 :找到 #Deployment ，修改为 

    # Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:yangzongjie/yangzongjie.github.io.git
  branch: master
  message:

  1. 验证主题是否启用

    $ hexo s –debug

    并访问 http://localhost:4000，确保站点正确运行。

    debug模式下，修改配置文件或博客内容时(command+s)，本地显示便即时更新（终端内安control+c退出debug）

  2. 设置pisces主题
    主题配置文件中，将 

    # Schemes
scheme: Muse
#scheme: Mist
#scheme: Pisces

    修改为 

    # Schemes
#scheme: Muse
#scheme: Mist
scheme: Pisces

    Schemes为NexT的子主题，可以任意选择

八 将本地数据部署至gitHub (每次修改配置或内容后都需执行以下命令，更新博客)

$ hexo clean   

$ hexo generate -deploy  
或者直接缩写
$ hexo g -d

九 进入yangzongjie.github.io就可以看到自己的博客啦！4不4SoEZ
