Git常用命令

发表于 2015-12-20   |   分类于 软件及网络应用

安装git

通过brew安装

切换为RubyGems 镜像源为淘宝。

$ gem sources --add https://ruby.taobao.org/ --remove https://rubygems.org/
$ gem sources -l
*** CURRENT SOURCES ***

https://ruby.taobao.org

$ gem install rails

安装Homebrew

$ ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

homebrew安装git(若装有xcode，则跳过该步骤)

$ brew install git

通过Xcode安装

启动并进入 Preferences -> Download -> Command Line Tools -> Install 安装命令行工具 .

git设置代理(无翻墙请跳过)

git config --global http.https://github.com.proxy socks5://127.0.0.1:1080

配置全局用户名，邮箱（仅用于显示、方便交流）

$ git config --global user.name "yangzongjie"  
$ git config --global user.email "340244120@qq.com"

所有项目默认使用，如需给项目单独配置，可进入路径，去掉global参数。

创建本地仓库(repository)

  1. 初始化仓库 

    $ git init

  2. 添加文件至暂存区(stage)

    $ git add xxx.m

  3. 提交暂存区文件至本地仓库

    $ git commit -m “add/modified…”

提交修改

  1. 查看已变更的文件的状态(not staged 或 to be committed) 

    $ git status

  2. 查看具体的变更内容 

    $ git diff

  3. 添加变更文件至stage 

    $ git add xxx.m

  4. 提交暂存区文件至 本地仓库 

    $ git commit -m "add/modified..."

时光穿梭机

版本回退

  1. 显示日志 

    $ git log (--pretty=oneline) // 显示提交日志 （回到过去后，日志的记忆也会丢失）
$ git reflog   // 显示完整的提交/回退日志（回到过去后，所有记忆保留 ）

  2. 版本回退 

    $ git reset --hard <commit hashID>（hashID在日志里为hashi值前7位）

  3. 撤销回退

    注意,提交回退时,log也同时回退.若要撤销回退,可利用git reflog命令查看所有操作记录. 

    $ git reflog
ea34578 HEAD@{0}: reset: moving to HEAD^
3628164 HEAD@{1}: commit: append GPL
ea34578 HEAD@{2}: commit: add distributed
cb926e7 HEAD@{3}: commit (initial): wrote a readme file 

$ git reset --hard <commit id>

管理修改

每次修改都必须添加到暂存区,才能commit.

如果某文件修改后添加到暂存区,又做了一次修改,扔必须git add添加到暂存区后才能提交这2次修改.

撤销修改

撤销当前工作区的修改(), 这里有2种情况 :
一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。 

$  git checkout -- <file>  // 用仓库里的版本覆盖工作区的版本

若要撤销已经提交到暂存区的修改,则 

$ git reset HEAD <file> // 撤销暂存区修
$ git checkout --<file> // 本地库覆盖工作区

HEAD为指向当前版本的指针,及最新的提交.上一个版本是HEAD\^,上上个版本HEAD\^\^\

删除文件

  • 确认删除
    工作区删除文件后,执行git rm后才能git commit. 

    $ git rm <file>  
$ git commit -m "delete <file>"

  • 撤销删除 

    $ git checkout -- <file> // 用仓库里的版本替换工作区的版本

添加远程仓库

添加远程库(先本地后远程)

  1. 创建SSH key（并向gitHub设置界面添加公钥）

    $ ssh-keygen -t rsa -C "yangzongjie@gmail.com"

    只有github里添加的公钥,才有资格提交文件至远程仓库.

  2. 将本地库与远程库关联

    $ git remote add origin git@github.com:yangzongjie/repositoryname.git

  3. 第一次将本地库内容推送至远程库

    $ git push -u origin master  
// 由于远程库是空的，第一次推送master分支时，加上了-u参数，
// Git不但会把本地的master分支内容推送的远程新的master分支，
// 还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

    从现在起，只要本地作了提交，就可以通过命令：

    $ git push origin master

    把本地master分支的最新修改推送至GitHub.

克隆远程库(先远程后本地，最常用、方便)

$ git clone git@github.com:yangzongjie/repositoryname.git

分支管理

创建与合并分支

  1. 创建并切换dev分支,

    $ git checkout -b dev
//$ git branch dev (创建dev分支)
//$ git checkout dev (切换dev分支)
Switched to a new branch 'dev'
// 然后我们的后续操作都默认在dev分支上进行

  2. 查看当前分支

    $ git branch
* dev
    master

  3. 切换至master分支

    $ git checkout master

  4. 将dev分支合并到master分支

    $ git merge dev  // 合并指定分支到当前分支,若无冲突则"快速合并"

  5. 删除dev分支

    $ git branch -d dev

    因为创建、合并和删除分支非常快，所以Git鼓励使用分支完成某个任务，合并后再删掉分支，这和直接在master分支上工作效果是一样的，但过程更安全

解决冲突(例如:新建featurel分支)

  1. 新建featurel分支

    $ git checkout -b feature1

  2. 修改文件后,切回至master

    $ git checkout master
Switched to branch 'master'
Your branch is ahead of 'origin/master' by 1 commit.

  3. 修改文件后,提交

    $ git add readme.txt 
$ git commit -m "& simple"
[master 400b400] & simple
 1 file changed, 1 insertion(+), 1 deletion(-)

  4. 若尝试合并

    $ git merge feature1
Auto-merging readme.txt
CONFLICT (content): Merge conflict in readme.txt
Automatic merge failed; fix conflicts and then commit the result.

  1. 查看冲突文件 git status

    $ git status
## On branch master
## Your branch is ahead of 'origin/master' by 2 commits.
##
## Unmerged paths:
##   (use "git add/rm <file>..." as appropriate to mark resolution)
##
##       both modified:      readme.txt
##
no changes added to commit (use "git add" and/or "git commit -a")

  2. 打开文件查看内容(readme.text)

    Git is a distributed version control system.
Git is free software distributed under the GPL.
Git has a mutable index called stage.
Git tracks changes of files.
<<<<<<< HEAD
Creating a new branch is quick & simple.
=======
Creating a new branch is quick AND simple.
>>>>>>> feature1

  3. 手动解决冲突后,提交

    $ git add readme.txt 
$ git commit -m "conflict fixed"
[master 59bc1cb] conflict fixed

  4. 查看分支合并情况git log

    $ git log --graph --pretty=oneline --abbrev-commit
*   59bc1cb conflict fixed
|\
| * 75a857c AND simple
* | 400b400 & simple
|/
* fec145a branch test
...

  5. 删除featurel分支

    $ git branch -d featurel
Deleted branch feature1 (was 75a857c).

Bug分支

  • 查看当前版本标签 
    `$ git tag`
  • 在本地代码库为项目打上标签
    $ git tag -a v1.0 -m"version 1.0"
  • 将标签推送至远程库
    $ git push origin v1.0



  • 使用tag。将项目快速切到某一版本，如切换到出现bug的1.0版本 
    `git checkout  v1.0`
  • 创建bug修复分支 
    `git checkout -b bugfix1.0`
  • 利用merge命令合并分支 
    `代码实现看具体情况`



  • 查看远程分支 
    `$ git branch -r`
  • 删除远程bug修复分支 
    `$ git branch -r -d origin/bugfix1.0`
